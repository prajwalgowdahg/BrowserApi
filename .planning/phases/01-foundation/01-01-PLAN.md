<task type="auto">
  <name>Task 2: Express app, middleware, health route, and server entry point</name>
  <files>src/utils/response.ts, src/middleware/errorHandler.ts, src/middleware/notFound.ts, src/routes/health.ts, src/app.ts, src/server.ts</files>
  <action>
1. Create src/utils/response.ts:
   - Define SuccessResponse<T> interface: { success: true; data: T }
   - Define ErrorResponse interface: { success: false; error: string; screenshot?: string }
   - Export function success<T>(res: Response, data: T, status = 200): Response
     - Returns res.status(status).json({ success: true, data } satisfies SuccessResponse<T>)
   - Export function error(res: Response, message: string, status = 500): Response
     - Returns res.status(status).json({ success: false, error: message } satisfies ErrorResponse)
   - IMPORTANT: Use res.status(code).json(obj) -- NOT res.json(obj, code). Express 5 removed the two-arg signature.

2. Create src/middleware/errorHandler.ts:
   - Export function errorHandler(err: Error, _req: Request, res: Response, _next: NextFunction): void
   - Log error: console.error('[ERROR]', err.message, err.stack)
   - Check res.headersSent -- if true, call _next(err) and return (prevents ERR_HTTP_HEADERS_SENT)
   - Extract status from err.status or default to 500
   - Extract message from err.message or default to 'Internal server error'
   - Return res.status(status).json({ success: false, error: message })

3. Create src/middleware/notFound.ts:
   - Export function notFoundHandler(_req: Request, res: Response): void
   - Return res.status(404).json({ success: false, error: 'Not found' })
   - IMPORTANT: This is a regular middleware (not error middleware -- 3 params, not 4)

4. Create src/routes/health.ts:
   - Import Router from 'express', success from '../utils/response'
   - Export const healthRouter = Router()
   - healthRouter.get('/') handler: call success(res, { status: 'ok' })
   - This is a synchronous handler -- no async needed

5. Create src/app.ts:
   - Import express, healthRouter, notFoundHandler, errorHandler
   - Export function createApp():
     - const app = express()
     - app.use(express.json()) -- MUST be first, before routes (Express 5 does not parse bodies by default)
     - app.use('/health', healthRouter)
     - app.use(notFoundHandler) -- after all routes, before error handler
     - app.use(errorHandler) -- MUST be last middleware
     - return app

6. Create src/server.ts:
   - Import { env } from './config/env' FIRST (triggers validation, crashes if invalid)
   - Import { createApp } from './app'
   - const app = createApp()
   - Add process-level safety nets AFTER app creation:
     - process.on('uncaughtException', ...) -- log '[FATAL] Uncaught exception:' + err, process.exit(1)
     - process.on('unhandledRejection', ...) -- log '[FATAL] Unhandled rejection:' + reason, process.exit(1)
   - app.listen(env.PORT, () => console.log('Server running on port', env.PORT))

Registration order in app.ts is CRITICAL: json parser -> routes -> 404 handler -> error handler. If error handler is registered before routes, it never catches anything. If 404 handler is before routes, it intercepts legitimate requests.

IMPORTANT: Do NOT import or use express-async-errors. Express 5 handles async errors natively. Do NOT wrap async handlers in try/catch + next(err). Just throw/reject and Express 5 forwards to error middleware automatically.
